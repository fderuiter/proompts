# Togaf Overview

## Prompts
- **[TOGAF Phase A - Architecture Vision](phase_a_vision.prompt.yaml)**: Guide for defining the Architecture Vision, stakeholders, and the Statement of Architecture Work (The Mandate).
- **[TOGAF Phase B - Business Architecture](phase_b_business.prompt.yaml)**: Guide for defining the Business Architecture, baseline/target states, and gap analysis (The Core Engine).
- **[TOGAF Phase C - Information Systems Architectures](phase_c_information_systems.prompt.yaml)**: Guide for defining the Information Systems Architectures (Data and Application Architectures).
- **[TOGAF Phase D - Technology Architecture](phase_d_technology.prompt.yaml)**: Guide for defining the Technology Architecture (infrastructure, hardware, networks).
- **[TOGAF Phase E - Opportunities & Solutions](phase_e_opportunities.prompt.yaml)**: Guide for identifying delivery vehicles (projects), grouping gaps into work packages, and creating the initial roadmap (The Strategic Bridge).
- **[TOGAF Phase F - Migration Planning](phase_f_migration.prompt.yaml)**: Guide for creating the detailed Implementation and Migration Plan and prioritizing work packages (Refined Planning).
- **[TOGAF Phase G - Implementation Governance](phase_g_governance.prompt.yaml)**: Guide for overseeing the implementation to ensure conformance with the architecture (Sustaining Integrity).
- **[TOGAF Phase H - Architecture Change Management](phase_h_change_management.prompt.yaml)**: Guide for managing the architecture lifecycle after deployment and handling change requests (The Living Entity).
- **[TOGAF Preliminary Phase](preliminary_phase.prompt.yaml)**: Guide for establishing the Architecture Capability and defining the organizational footprint (The Genesis of Architecture).
- **[TOGAF Requirements Management](requirements_management.prompt.yaml)**: Guide for the continuous process of managing architecture requirements throughout the ADM cycle (The Dynamic Core).
