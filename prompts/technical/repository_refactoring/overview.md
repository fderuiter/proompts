# Repository Refactoring Overview

## Prompts
- **[Code Formatting, Linting, and Refactoring Implementation](code_formatting_linting_refactoring_implementation.prompt.yaml)**: Improve the codebase's internal quality and consistency by introducing and configuring a standard code formatter and a linter, and refactoring complex code.
- **[Codebase Quality & Maintainability Analysis](codebase_quality_analysis.prompt.yaml)**: Conduct a deep analysis of the codebase's quality and maintainability to identify key areas for refactoring.
- **[Dependencies & Security Posture Analysis](dependencies_security_analysis.prompt.yaml)**: Perform a thorough audit of the repository's dependencies and overall security posture to identify and mitigate risks.
- **[Documentation and Repository Structure Implementation](documentation_structure_implementation.prompt.yaml)**: Implement foundational improvements for the repository's structure and documentation.
- **[Repository Foundation & Developer Experience Analysis](repo_foundation_analysis.prompt.yaml)**: Analyze the repository's foundation and developer experience to prepare it for future growth and easy onboarding.
- **[Security Hardening and Dependency Management Implementation](security_hardening_dependency_management_implementation.prompt.yaml)**: Secure the repository and manage its dependencies by externalizing secrets, addressing vulnerabilities, and updating dependencies.
- **[Test Suite Enhancement and CI Pipeline Implementation](test_suite_enhancement_ci_pipeline_implementation.prompt.yaml)**: Build the automated quality gates for this repository by increasing test coverage, adding meaningful unit tests, and introducing a basic CI pipeline.
- **[Testing, Configuration, and Automation Analysis](testing_configuration_automation_analysis.prompt.yaml)**: Analyze the repository's testing, configuration, and automation infrastructure to ensure reliability and deployment readiness.
