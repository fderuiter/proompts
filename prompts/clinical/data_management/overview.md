# Data Management Overview

## Categories
- [**Cdisc Compliance Workflow**](cdisc_compliance_workflow/overview.md)
- [**Data Management Etl Workflow**](data_management_etl_workflow/overview.md)

## Prompts
- **[Audit Trail Review](audit_trail_review.prompt.yaml)**: Review subject audit logs for compliance and data integrity.
- **[CDISC CRF Architect](cdisc_crf_architect.prompt.yaml)**: Design CDASH/SDTM compliant CRFs from a Clinical Protocol.
- **[CDISC SDTM/ADaM Mapping](cdisc_mapping.prompt.yaml)**: Map raw clinical data to standardized CDISC SDTM and ADaM domains.
- **[Data Architecture Blueprint](data_architecture_blueprint.prompt.yaml)**: Draft a blueprint for clinical data architecture.
- **[Data De-identification](data_deidentification.prompt.yaml)**: De-identify patient-level data according to HIPAA Privacy Rule.
- **[Database Lock Procedures](database_lock_procedures.prompt.yaml)**: Review CRFs and prepare for database lock.
- **[Decentralized Trial Risk Matrix](decentralized_trial_risk_matrix.prompt.yaml)**: Build a risk matrix for decentralized trials.
- **[Data Management Plan (DMP) Development](dmp_development.prompt.yaml)**: Create a DMP outlining data lifecycle and quality control.
- **[Clinical Trial Document Archiving](document_archiving.prompt.yaml)**: Develop archival strategy for TMF.
- **[eTMF Artifact Classifier](etmf_artifact_classifier.prompt.yaml)**: Read document text and suggest appropriate eTMF artifact classification and metadata assignments for incoming trial documents.
- **[Medical Coding and Reconciliation Assistant](medical_coding_reconciliation.prompt.yaml)**: Automatically predict and apply medical terms to clinical data, and perform automated data reconciliation and query resolution within EDC builds.
- **[Metadata Management](metadata_management.prompt.yaml)**: Extract and store standardized metadata for reuse.
- **[21 CFR Part 11 Compliance Verification](part_11_compliance_verification.prompt.yaml)**: Confirm system compliance with electronic signatures.
- **[Phase II Oncology DMP](phase_ii_oncology_dmp.prompt.yaml)**: Create a Data Management Plan for a Phase II oncology study.
- **[Regulatory Compliance Verification](regulatory_compliance_verification.prompt.yaml)**: Verify electronic records and signatures against 21 CFR Part 11.
- **[Regulatory Gap Analysis](regulatory_gap_analysis.prompt.yaml)**: Assess regulatory compliance gaps in trial data processes.
- **[SOP Gap Analysis](sop_gap_analysis.prompt.yaml)**: Identify gaps in data management standard operating procedures.
- **[Trial Master File (TMF) Maintenance](tmf_maintenance.prompt.yaml)**: Generate TMF checklist based on CDISC Reference Model.
- **[Unified Data Cleansing](unified_data_cleansing.prompt.yaml)**: Outline a unified data cleansing approach for clinical trial datasets with strict CDISC/SDTM compliance.
